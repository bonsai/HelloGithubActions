# SF落語：nandoki.yml ってぇのは何時だい

えー、GitHub Actions でございます。
宙町の長屋、クマエモンが言う。

「ご隠居、いま何時で？」

するとご隠居、端末を指してこう言う。

「それを聞く相手が違う。聞くなら Actions に聞け」

## これが当の nandoki.yml で

場所はこれです。

- `.github/workflows/nandoki.yml`

こいつは“時刻”と“状況”を喋る。
何が起きたか、誰が起こしたか、どの枝で、どのハッシュで――そういうのを、いちいち手で確認しない。

## いつ動くんで？

クマエモンが聞く。

「ご隠居、勝手に喋りだしたら困りますよ」

ご隠居が扇子でトントン。

- `workflow_dispatch`：手動で「今だ！」って押した時  
- `pull_request`：main に向けた PR が出入りした時  
- `schedule`：毎時00分（UTC）に鳴る時報

時報ってのは便利でしてね、動いてるかどうかが分かる。
“動いてる証拠”が残るんです。

## 何を喋るんで？

「時刻だけかい？」って言うと、違う。
時刻に“札”が付くんです。

- `utc`：世界の時計（UTC）
- `local`：走ってる機械の時計（runnerのタイムゾーン）
- `run_id` / `run_number`：今回の番号札
- `event`：なんの用件で起きたか
- `ref` / `sha`：どの枝、どのコミットか
- `actor`：誰が起こしたか
- `repo`：どの家（リポジトリ）か

ご隠居が言う。

「ログはな、“読む”んじゃない。“照らす”んだ。いまを見えるようにする」

## 見る場所（実務）

1. GitHub の Actions タブを開く
2. `nandoki` を選ぶ
3. 実行ログと Summary を見る

まとめ（Summary）には、時刻と状況が短く並びます。
忙しい時ほど、短いのが助かる。

## 注意（小言）

- `schedule` の `cron` は UTC で動きます
- `local` はあなたのPCの時刻じゃなくて、runner の時刻です

えー、そんな宙町の時報でございました。
おあとがよろしいようで。
